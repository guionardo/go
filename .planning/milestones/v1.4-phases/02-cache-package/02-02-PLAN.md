**cache/redis/redis.go** — Redis Cache[K,V] implementation following RESEARCH.md lines 539-619 pattern:
- Package: `redis`
- Import: `context`, `encoding/json`, `fmt`, `time`, `"github.com/redis/go-redis/v9"`, `"github.com/guionardo/go/cache"`
- Struct `Cache[K comparable, V any]`:
  - `client *redis.Client`
  - `defaultTTL time.Duration`
- Constructor `func New[K comparable, V any](opts ...Option) *Cache[K, V]`:
  - `cfg := defaultConfig(); for _, opt := range opts { opt(cfg) }`
  - `client := redis.NewClient(&redis.Options{Addr: cfg.Addr, Password: cfg.Password, DB: cfg.DB, PoolSize: cfg.PoolSize})`
  - Return `&Cache[K, V]{client: client, defaultTTL: cfg.DefaultTTL}`
- Methods:
  - `Get(ctx context.Context, key K) (V, error)`:
    - `data, err := c.client.Get(ctx, fmt.Sprint(key)).Bytes()`
    - If `err == redis.Nil`: return zero + `fmt.Errorf("cache/redis: %w", cache.ErrMiss)`
    - If `err != nil`: return zero + `fmt.Errorf("cache/redis: %w", err)`
    - `json.Unmarshal(data, &value)` — if error: return zero + `fmt.Errorf("cache/redis: %w", err)`
    - Return `value, nil`
  - `Set(ctx context.Context, key K, value V, ttl ...time.Duration) error`:
    - `data, err := json.Marshal(value)` — if error: `fmt.Errorf("cache/redis: %w", err)`
    - `expiration := c.resolveTTL(ttl...)`
    - `err = c.client.Set(ctx, fmt.Sprint(key), data, expiration).Err()` — if error: `fmt.Errorf("cache/redis: %w", err)`
    - Return nil
  - `Delete(ctx context.Context, key K) error`:
    - `err := c.client.Del(ctx, fmt.Sprint(key)).Err()` — if error: `fmt.Errorf("cache/redis: %w", err)`
    - Return nil
  - `GetOrSet(ctx context.Context, key K, setter func() (V, error), ttl ...time.Duration) (V, error)`:
    - First try Get; if no error, return value
    - If error: call setter(); if setter errors, return zero + wrapped error
    - Call Set with computed value and ttl
    - Return value, nil
  - `Close() error`: return `c.client.Close()` — wrap with `fmt.Errorf("cache/redis: %w", err)` if error
  - `resolveTTL(ttl ...time.Duration) time.Duration`:
    - If `len(ttl) > 0 && ttl[0] > 0`: return `ttl[0]`
    - If `c.defaultTTL > 0`: return `c.defaultTTL`
    - Return `0` (no expiry — Redis interprets 0 as no expiry)
- All errors wrapped with `"cache/redis: %w"` per D-12
- Key conversion: `fmt.Sprint(key)` for all external calls

- Constructor `func New[K comparable, V any](opts ...Option) *Cache[K, V]`:
  - `cfg := defaultConfig(); for _, opt := range opts { opt(cfg) }`

    ```go
    client, err := valkey.NewClient(valkey.ClientOption{
        Addr:     cfg.Addr,
        Password: cfg.Password,
        SelectDB: cfg.DB,
    })
    ```
    If err, panic with meaningful message (constructor contract violation pattern from config/options.go — panics for unrecoverable config errors)
  - Return `&Cache[K, V]{client: client, defaultTTL: cfg.DefaultTTL}`
- Methods:
  - `Get(ctx context.Context, key K) (V, error)`:
    - Valkey-go API uses `client.Do(ctx, client.B().Get().Key(fmt.Sprint(key)).Build()).ToString()`
    - If error (not found): check if error is valkey.Nil — return zero + `fmt.Errorf("cache/valkey: %w", cache.ErrMiss)`
    - Other errors: return zero + `fmt.Errorf("cache/valkey: %w", err)`
    - `json.Unmarshal([]byte(data), &value)` — if error: `fmt.Errorf("cache/valkey: %w", err)`
    - Return `value, nil`
  - `Set(ctx context.Context, key K, value V, ttl ...time.Duration) error`:
    - `data, err := json.Marshal(value)` — if error: `fmt.Errorf("cache/valkey: %w", err)`
    - `expiration := c.resolveTTL(ttl...)`
    - Build Set command: `client.Do(ctx, client.B().Set().Key(fmt.Sprint(key)).Value(string(data)).Ex(expiration).Build()).Error()`
    - If error: `fmt.Errorf("cache/valkey: %w", err)`
    - Return nil
  - `Delete(ctx context.Context, key K) error`:
    - `err := client.Do(ctx, client.B().Del().Key(fmt.Sprint(key)).Build()).Error()`
    - If error: `fmt.Errorf("cache/valkey: %w", err)`
    - Return nil
  - `GetOrSet(ctx context.Context, key K, setter func() (V, error), ttl ...time.Duration) (V, error)`: Same pattern as Redis — try Get, on error call setter, Set result
  - `Close() error`: return `c.client.Close()` — wrap with `fmt.Errorf("cache/valkey: %w", err)` if error
  - `resolveTTL(ttl ...time.Duration) time.Duration`: Same logic as Redis — return 0 for no expiry
- All errors wrapped with `"cache/valkey: %w"` per D-12

**Important valkey-go API notes:**
- Valkey-go uses a builder pattern for commands: `client.B().Get().Key("key").Build()`
- Set with TTL: `client.B().Set().Key("key").Value("value").Ex(ttl).Build()`
- Error handling: `resp, err := client.Do(ctx, cmd).ToString()` for string responses
- The `Ex(ttl)` parameter takes time.Duration; use 0 for no expiry
- Check for nil (key not found): valkey-go returns `valkey.Nil` error

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Consumer → Redis/Valkey Cache provider | Caller's code passes typed K,V parameters; key is converted to string via fmt.Sprint |
| Redis/Valkey provider → External server | TCP connection to Redis/Valkey server — network boundary |
| JSON serialization/deserialization | Value serialization via encoding/json — type-safe at compile time |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Tampering | Key injection | medium | accept | Keys converted with fmt.Sprint(key) only — no raw string construction. Cache keys are caller-controlled already. |
| T-02-04 | Information Disclosure | Connection secrets (password) | high | mitigate | Password accepted via functional option `WithPassword(string)` — recommend callers source from environment variables, not hardcoded values. The option pattern avoids embedding secrets in connection strings. |
| T-02-05 | Tampering | JSON deserialization | low | accept | encoding/json is safe against arbitrary data; generic type parameter V ensures type safety at compile time. |
| T-02-06 | Spoofing | Connection to wrong Redis/Valkey server | medium | accept | Address configurable via WithAddr option; caller responsibility to configure correct endpoint. |
</threat_model>
