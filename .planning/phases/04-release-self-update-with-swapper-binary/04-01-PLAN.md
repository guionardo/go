    ```go
    package release

    // Version represents a parsed semantic version (v{major}.{minor}.{patch}[-{prerelease}]).
    // The leading "v" prefix is stripped if present.
    type Version struct {
        Major      int
        Minor      int
        Patch      int
        Prerelease string // Empty if no prerelease suffix
    }

    // ParseVersion parses a version string like "v1.2.3", "1.2.3", or "v1.2.3-beta.1".
    // Returns an error if the string does not match semver format.
    func ParseVersion(s string) (Version, error)

    // String returns the canonical string representation (e.g., "v1.2.3", "v1.2.3-beta.1").
    func (v Version) String() string

    // Compare returns -1 if v &lt; other, 0 if equal, 1 if v &gt; other.
    // Comparison follows semver precedence rules: major &gt; minor &gt; patch.
    // Versions with prerelease are considered lower than the same version without (per semver spec §11).
    func (v Version) Compare(other Version) int
    ```

    Implementation rules:

    1. **ParseVersion**: Strip leading "v" prefix using `strings.TrimPrefix(s, "v")`. Split on `.`. Parse major, minor, patch with `strconv.Atoi`. If a `-` exists after patch, split on `-` and store the remainder as Prerelease. Validate that all three numeric parts parse successfully. Return descriptive error messages.

    2. **Compare**: Compare major first, then minor, then patch. If patch is equal and both have prerelease, compare prerelease strings lexicographically (per semver spec §11: numeric identifiers compare numerically, alphanumeric compare ASCII — shell out to `github.com/Masterminds/semver` would be overkill; use `strings.Compare` for simplicity. This is sufficient for the common case of `-beta.1`, `-rc.2`, etc.). If patch is equal and one has prerelease and the other does not, the prerelease version is lower (semver spec §11 rule).

    3. **String**: Always emit with "v" prefix. Format as `v{Major}.{Minor}.{Patch}`. Append `-{Prerelease}` if non-empty.

    4. **GetCurrentVersion**: Add a package-level function that reads build info via `debug.ReadBuildInfo()` (same pattern as `getCurrentModule` in `release.go`). Extract the module version from the main module's version field. Parse it with `ParseVersion` and return. If build info is unavailable, return a zero Version with an error.

    ```go
    func GetCurrentVersion() (Version, error)
    ```

    Test file `release/version_test.go`:

    - TestParseVersion_Valid: "v1.2.3", "1.2.3", "v0.0.1", "v10.20.30", "v1.2.3-beta.1", "v1.2.3-rc2"
    - TestParseVersion_Invalid: "", "v1", "v1.2", "abc", "v1.2.3.4"
    - TestCompare: v1.2.3 &gt; v1.2.2, v2.0.0 &gt; v1.99.99, v1.2.3-beta &lt; v1.2.3, v1.2.3 == v1.2.3, v1.2.3-alpha &lt; v1.2.3-beta (lexicographic prerelease)
    - TestString: check round-trip ParseVersion(s).String() == s (with "v" prefix normalization)
    - TestGetCurrentVersion: verify it returns a version (integration test that reads build info — cannot run in unit test context; use a table-driven test that checks behavior without relying on build info, plus note that real integration requires `go test` with module build)

    Use `testing` stdlib with `assert`/`require` from `github.com/stretchr/testify` (consistent with project convention — see `release_test.go`). Tests run in parallel per project pattern (`t.Parallel()`).

    ## release/update.go — Update detection and download API

    ```go
    package release

    // Config holds configuration for update detection.
    type Config struct {
        Owner string // GitHub owner (overrides auto-detection)
        Repo  string // GitHub repo (overrides auto-detection)
        Token string // GitHub API token for authenticated requests (higher rate limit)
    }
    ```

    Define an `Option` interface with `Apply(*Config)` method, and `optionFunc` adapter type — consistent with the `cache` package pattern (see `cache/options.go`).

    Implement functional options:

    - `WithOwner(owner string) Option` — sets Config.Owner
    - `WithRepo(repo string) Option` — sets Config.Repo
    - `WithGitHubToken(token string) Option` — sets Config.Token

    Implementation:

    ```go
    // CheckForUpdate queries GitHub for the latest release of the current module
    // and returns it along with a boolean indicating if it's newer than currentVersion.
    // If currentVersion is empty, the release is always considered "newer".
    // Uses auto-detection (debug.ReadBuildInfo) by default; override with functional options.
    func CheckForUpdate(ctx context.Context, currentVersion string, opts ...Option) (*Release, bool, error)
    ```

    1. Apply options to a new Config
    2. If Config.Owner and Config.Repo are set, call `GetLatestRelease(owner, repo)`
    3. Otherwise call `GetThisLatestRelease()` (auto-detection per D-08)
    4. If currentVersion is empty, return (release, true, nil)
    5. Parse currentVersion with ParseVersion — if parse fails, return error
    6. Parse release.TagName with ParseVersion — if parse fails, skip version check (treat as newer) with a logged warning
    7. Compare: if release version > current, return (release, true, nil); otherwise (release, false, nil)
    8. Pass context for cancellation if GitHub API supports it (use context-aware HTTP if possible, else use existing githubClient)

    ```go
    // DownloadUpdate downloads the release asset matching the current GOOS/GOARCH,
    // verifies its checksum, and returns the path to the verified binary on disk.
    // The binary is written to targetDir with the same filename as the asset.
    func DownloadUpdate(ctx context.Context, rel *Release, targetDir string) (string, error)
    ```

    1. Find the asset for `runtime.GOOS` and `runtime.GOARCH` by checking `Asset.Name` for platform suffix patterns like `linux_amd64`, `darwin_arm64`, `windows_amd64.exe`, etc. Return error if no matching asset.
    2. Create a temp file in `targetDir` using `os.CreateTemp(targetDir, ".download-*")` — this ensures same-filesystem rename (per Pitfall 1 in RESEARCH.md).
    3. Download the asset using `asset.Download(tmpFile)` — this already verifies the go-digest per existing code in release.go.
    4. Close the temp file.
    5. If checksums.txt is available in the release, download it and verify the asset against it.
    6. Rename the temp file to the final asset name in targetDir.
    7. Return the final path.

    ```go
    // findAsset searches release assets for one matching the given GOOS/GOARCH.
    // Returns nil if no match found.
    func findAsset(rel *Release, goos, goarch string) *Asset
    ```

    Matching logic: check if `Asset.Name` contains both goos and goarch substrings. For Windows, also match `.exe` suffix. Handle common naming patterns like `binary_v1.2.3_linux_amd64.tar.gz`, `binary_darwin_arm64.zip`, etc.

    ## release/checksum.go — Checksum verification utilities

    ```go
    package release

    // VerifyChecksum verifies that filePath's SHA256 hex digest matches expectedHex.
    // Uses crypto/sha256, NOT go-digest, for the checksums.txt verification path.
    func VerifyChecksum(filePath string, expectedHex string) error
    ```

    1. Read file with `os.ReadFile`
    2. Compute `sha256.Sum256(data)`
    3. Encode as hex via `hex.EncodeToString`
    4. Use `subtle.ConstantTimeCompare` for timing-safe comparison (per ASVS V6 Cryptography)
    5. Return descriptive error on mismatch

    ```go
    // ParseChecksumsFile parses a standard checksums.txt file (format: "<hex>  <filename>").
    // Returns a map from filename to hex checksum.
    func ParseChecksumsFile(r io.Reader) (map[string]string, error)
    ```

    ```go
    // VerifyFromChecksumsFile looks up filePath's basename in the checksums file
    // and verifies the file's SHA256 matches. Returns error if filename not found
    // or checksum mismatch.
    func VerifyFromChecksumsFile(filePath string, checksumsFile string) error
    ```

| T-04-04 | Information Disclosure | checksums.txt download | low | accept | Checksums.txt is public release artifact; no sensitive data exposed. |
| T-04-SC | Tampering | Package installs | low | accept | No new external packages installed. All functionality uses Go stdlib + existing go-digest dependency. No Package Legitimacy Gate needed per RESEARCH.md audit table. |
