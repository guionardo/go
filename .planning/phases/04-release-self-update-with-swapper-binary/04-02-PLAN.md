<objective>
Create the standalone swapper binary that performs atomic binary replacement with backup/rollback and cross-platform relaunch.

**Purpose:** The swapper is a compiled binary embedded in the main binary via `//go:embed` (Plan 3). It runs as a separate process after the main binary exits, bypassing Windows file-locking issues. It handles the critical safety guarantees: backup old binary, copy new binary, re-verify checksum, restore on failure, and relaunch with original arguments.

**Output:** A `package main` under `release/swapper/` with entry point, atomic swap logic, and platform-tagged exec files. All with tests.

    ## release/swapper/swap.go — Core swap logic

    Package declaration: `package main`

    ```go
    // atomicReplace atomically replaces target with replacement using a backup-then-swap
    // strategy. On any error, it attempts to restore the backup before returning.
    //
    // Flow:
    //   1. Rename target → target.bak (backup)
    //   2. Rename replacement → target (swap)
    //   3. Remove target.bak (cleanup)
    //   4. If step 1 fails → return error (no backup created, nothing to restore)
    //   5. If step 2 fails → restore target.bak → target, return error
    //   6. If step 3 fails → non-fatal (backup orphaned, swap succeeded)
    func atomicReplace(target, replacement string) error
    ```

    Implementation rules:
    1. Before any rename, call `filepath.EvalSymlinks` on target to resolve any symlinks (mitigates symlink-race attacks per T-04-06).
    2. Backup: `os.Rename(target, target+".bak")`. On error, return wrapped error immediately.
    3. Swap: `os.Rename(replacement, target)`. On error, attempt `os.Rename(target+".bak", target)` to restore. Return wrapped error with note that backup was restored.
    4. Cleanup: `os.Remove(target+".bak")`. On error, log via `fmt.Fprintf(os.Stderr, ...)` but do NOT return error — the swap succeeded.
    5. Use `fmt.Errorf("backup %s -&gt; %s.bak: %w", target, target, err)` for error wrapping with `%w` so callers can use `errors.Is`/`errors.As`.

    ```go
    // verifyChecksum computes SHA256 of filePath and compares against expectedHex.
    // Uses subtle.ConstantTimeCompare for timing-safe comparison.
    func verifyChecksum(filePath, expectedHex string) error
    ```

    1. Read file with `os.ReadFile`
    2. Compute `sha256.Sum256(data)`
    3. `hex.EncodeToString(sum[:])`
    4. Compare with `crypto/subtle.ConstantTimeCompare`
    5. Return descriptive error on mismatch

    ## release/swapper/swap_windows.go — Windows process spawn (build-tag: windows)

    ```go
    //go:build windows

    package main

    import "os"

    // relaunch starts the new binary as a child process, then exits the current process.
    // On success, this function never returns (current process exits).
    // On failure, it returns an error.
    func relaunch(execPath string, args, env []string) error {
        proc, err := os.StartProcess(execPath, args, &os.ProcAttr{
            Env:   env,
            Files: []*os.File{os.Stdin, os.Stdout, os.Stderr},
        })
        if err != nil {
            return err
        }
        if err := proc.Release(); err != nil {
            return err
        }
        os.Exit(0)
        return nil
    }
    ```

    Create `release/swapper/main.go`:

    ```go
    // Command swapper performs atomic binary replacement and relaunch.
    //
    // Usage (invoked by parent binary, not directly by user):
    //
    //   swapper --new-binary=<path> [--checksum=<sha256>] [original args...]
    //
    // Flow per D-05:
    //   1. Parse --new-binary flag
    //   2. Resolve current executable path
    //   3. Verify SHA256 of new binary (second verification phase per D-01)
    //   4. Backup old binary and swap new into place
    //   5. Re-verify the swapped binary on disk
    //   6. Relaunch with original arguments
    //
    // Exit codes:
    //   0 — Success (relaunch never returns on success)
    //   1 — Any error (checksum mismatch, swap failure, etc.)
    package main

    import (
        "crypto/sha256"
        "encoding/hex"
        "flag"
        "fmt"
        "os"
        "path/filepath"
    )

    func main() {
        // Parse flags
        var newBinary string
        var expectedChecksum string
        flag.StringVar(&newBinary, "new-binary", "", "Path to the new binary (required)")
        flag.StringVar(&expectedChecksum, "checksum", "", "Expected SHA256 hex checksum for verification")
        flag.Parse()

        if newBinary == "" {
            fmt.Fprintln(os.Stderr, "Usage: swapper --new-binary=<path> [--checksum=<sha256>] [original args...]")
            os.Exit(1)
        }

        // Validate newBinary path — reject paths with ".." components or null bytes (mitigation for T-04-05)
        if strings.Contains(newBinary, "..") || strings.Contains(newBinary, "\x00") {
            fmt.Fprintln(os.Stderr, "invalid --new-binary path: path traversal detected")
            os.Exit(1)
        }

        // Resolve current executable path with symlink resolution
        currentExe, err := os.Executable()
        if err != nil {
            fmt.Fprintf(os.Stderr, "failed to get executable path: %v\n", err)
            os.Exit(1)
        }
        currentExe, err = filepath.EvalSymlinks(currentExe)
        if err != nil {
            fmt.Fprintf(os.Stderr, "failed to resolve symlinks: %v\n", err)
            os.Exit(1)
        }

        // Phase 1: Verify checksum of new binary (second line of defense per D-01)
        if expectedChecksum != "" {
            if err := verifyChecksum(newBinary, expectedChecksum); err != nil {
                fmt.Fprintf(os.Stderr, "pre-swap checksum verification failed: %v\n", err)
                os.Exit(1)
            }
        }

        // Phase 2: Atomic swap with backup
        if err := atomicReplace(currentExe, newBinary); err != nil {
            fmt.Fprintf(os.Stderr, "swap failed: %v\n", err)
            os.Exit(1)
        }

        // Phase 3: Re-verify the swapped binary on disk (per D-01)
        if expectedChecksum != "" {
            if err := verifyChecksum(currentExe, expectedChecksum); err != nil {
                fmt.Fprintf(os.Stderr, "post-swap verification failed: %v\n", err)
                // Attempt to restore backup before exiting
                backupPath := currentExe + ".bak"
                if _, statErr := os.Stat(backupPath); statErr == nil {
                    if renErr := os.Rename(backupPath, currentExe); renErr != nil {
                        fmt.Fprintf(os.Stderr, "backup restore also failed: %v\n", renErr)
                    }
                }
                os.Exit(1)
            }
        }

        // Phase 4: Relaunch with original args
        originalArgs := flag.Args()
        relaunchArgs := append([]string{currentExe}, originalArgs...)
        relaunch(currentExe, relaunchArgs, os.Environ())
        // Never returns on success
    }
    ```

    Implementation rules:

    1. **Flag parsing**: Use `flag` stdlib package. `--new-binary` is required. `--checksum` is optional (but strongly recommended per D-01). Remainder of args after flags are the original program arguments.

    2. **Path validation (T-04-05 mitigation)**: Check `newBinary` for `".."` and null byte (`"\x00"`) characters. Reject with clear error and exit code 1. Use `strings.Contains` from `"strings"` package.

    3. **Executable resolution**: `os.Executable()` returns the path to the current binary. `filepath.EvalSymlinks` resolves any symlink layers to get the real path (mitigates symlink-race attacks per T-04-06).

    4. **Error reporting (per D-13)**: All errors go to `os.Stderr`. Exit code 0 means success (never reached — relaunch replaces the process). Exit code 1 means failure.

    5. **Cleanup note**: On success, the backup file (`currentExe + ".bak"`) is removed in `atomicReplace`. On post-swap verification failure, an attempt is made to restore the backup before exiting with code 1. If backup restore also fails, both paths are reported to stderr.

    6. **Import `strings`**: Add `"strings"` to imports for the path traversal check.

    **Do NOT add tests for main.go separately** — the core logic (verifyChecksum, atomicReplace) is tested in Task 1. The `main` function is pure orchestration of already-tested primitives.

    **Build tag consistency:** `main.go` must compile on all platforms. No build tag on this file. The platform-specific code is in `swap_unix.go` (build tag `!windows`) and `swap_windows.go` (build tag `windows`).

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swapper process boundary | Untrusted data from --new-binary flag crosses here |
| Disk → Process | Binary file on disk is read and exec'd — symlink race vector |
| Backup file on disk | Leftover .bak files could be exploited by local attacker |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05 | Elevation of Privilege | main.go flag parsing | high | mitigate | Reject `--new-binary` paths containing `..` or null bytes. Use `filepath.EvalSymlinks` on target path. |
| T-04-06 | Elevation of Privilege | swap.go atomicReplace | high | mitigate | Resolve symlinks via `filepath.EvalSymlinks` before any rename operation. Create temp files in same directory as target (prevents cross-device symlink race). |
| T-04-07 | Elevation of Privilege | backup file handling | medium | mitigate | Remove backup file immediately after successful swap. On failure, restore backup before exit. Backup uses `.bak` extension with restrictive permissions inherited from original. |
| T-04-08 | Information Disclosure | verifyChecksum | low | mitigate | Use `subtle.ConstantTimeCompare` for checksum comparison to prevent timing side-channel attacks. |
| T-04-09 | Tampering | Concurrent swap operations | low | accept | Lock file mechanism deferred — swapper runs as short-lived process; concurrent invocation by the same user is self-correcting (second run sees original binary unchanged after first swap). Noted in security doc. |

</threat_model>
